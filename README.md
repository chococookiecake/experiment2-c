# 任务三：⽤户活跃度分析  

## 设计思路   
第一个任务的mapper：输入：userId 当日直接购买量 当日赎回量  ；输出：userId （当日活动？1：0）  
第一个任务的reducer：输入：userId，1，1，0，1，1，0，...   ；输出：userId 活动天数  
第二个任务中，mapper的输出以活动天数作为key，利用shuffle&sort阶段进行自动排序
